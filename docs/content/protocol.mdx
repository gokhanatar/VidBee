VidDownloadPro registers a custom URL protocol (`viddownloadpro://`) that allows you to trigger downloads directly from web browsers, browser extensions, or userscripts.

## Basic Usage

The `viddownloadpro://` protocol can be used to open VidDownloadPro and automatically start downloading videos.

### Protocol Format

```
viddownloadpro://download?url=<encoded-video-url>
```

**Parameters:**
- `url` (required): The video URL to download, must be URL-encoded

### Example

To download a YouTube video:

```html
<a href="viddownloadpro://download?url=https%3A%2F%2Fwww.youtube.com%2Fwatch%3Fv%3DdQw4w9WgXcQ">
  Download with VidDownloadPro
</a>
```

Or in JavaScript:

```javascript
const videoUrl = 'https://www.youtube.com/watch?v=dQw4w9WgXcQ'
const viddownloadproUrl = `viddownloadpro://download?url=${encodeURIComponent(videoUrl)}`
window.location.href = viddownloadproUrl
```

## Opening VidDownloadPro

To simply open the VidDownloadPro app without starting a download:

```
viddownloadpro://
```

## Use Cases

### Browser Extension

The VidDownloadPro browser extension uses this protocol to send the current tab's URL to the desktop app:

```javascript
const currentUrl = window.location.href
const deepLink = `viddownloadpro://download?url=${encodeURIComponent(currentUrl)}`
window.location.href = deepLink
```

### Userscript Integration

The VidDownloadPro userscript adds quick download buttons to supported video sites:

```javascript
// Single click triggers download via protocol
const viddownloadproUrl = `viddownloadpro://download?url=${encodeURIComponent(videoUrl)}`
window.location.href = viddownloadproUrl
```

### Web Pages

You can add direct download links to your web pages:

```html
<!-- Simple link -->
<a href="viddownloadpro://download?url=https%3A%2F%2Fexample.com%2Fvideo">
  Download with VidDownloadPro
</a>

<!-- Button with JavaScript -->
<button onclick="openInVidDownloadPro('https://example.com/video')">
  Quick Download
</button>

<script>
function openInVidDownloadPro(url) {
  const viddownloadproUrl = `viddownloadpro://download?url=${encodeURIComponent(url)}`
  window.location.href = viddownloadproUrl
}
</script>
```

## Playlist Support

To download an entire playlist:

```
viddownloadpro://download?url=<encoded-playlist-url>&type=playlist
```

**Parameters:**
- `url` (required): The playlist URL, must be URL-encoded
- `type`: Set to `playlist` to download all videos in the playlist

### Example

```javascript
const playlistUrl = 'https://www.youtube.com/playlist?list=PLrAXtmErZgOeiKm4sgNOknGvNjby9efdf'
const viddownloadproUrl = `viddownloadpro://download?url=${encodeURIComponent(playlistUrl)}&type=playlist`
window.location.href = viddownloadproUrl
```

## How It Works

1. **Protocol Registration**: VidDownloadPro registers as the handler for the `viddownloadpro://` protocol during installation
2. **URL Parsing**: When a `viddownloadpro://download?url=...` link is clicked, the OS launches VidDownloadPro
3. **Queue Processing**: VidDownloadPro extracts the video URL and adds it to the download queue
4. **Auto-start**: The download begins automatically if the app is configured for auto-download

## Browser Compatibility

The `viddownloadpro://` protocol works across all major browsers:
- Chrome/Edge/Brave
- Firefox
- Safari

## Security Notes

- Only URLs starting with `viddownloadpro://` will trigger the app
- The app validates the URL format before processing
- Malformed URLs are ignored with a warning in the logs

## Troubleshooting

### Protocol Not Working

If clicking `viddownloadpro://` links doesn't open VidDownloadPro:

1. **Check installation**: Ensure VidDownloadPro is properly installed
2. **Reinstall**: Try reinstalling VidDownloadPro to re-register the protocol
3. **OS permissions**: On macOS, check System Settings > Privacy & Security for any blocks
4. **Browser settings**: Some browsers may require you to allow the protocol on first use

### App Opens But Doesn't Download

If VidDownloadPro opens but the download doesn't start:

1. **Check URL encoding**: Ensure the video URL is properly encoded with `encodeURIComponent()`
2. **Check logs**: Open the app and check the developer console for errors
3. **Supported sites**: Verify the URL is from a [supported site](https://viddownloadpro.org/supported-sites/)
